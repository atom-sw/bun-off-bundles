# Documentation style

Write documentation and comments in a clear, consistent house style.

- Prefer the active voice and direct sentences: write "The parser validates the input,"
  not "The input is validated by the parser."
- Prefer colons over dashes to introduce an explanation, a list, or a definition. Reserve
  the en-dash (–) for a numeric range, such as "pages 10–25". Reserve the em-dash (—) for
  a strong interruption or aside, and reach for it rarely.
- Use an arrow (→) only to show a sequence or a hierarchy: the stages of a pipeline, or
  menu levels such as File → Save. Do not use an arrow as shorthand for a chain of
  reasoning: write the implication out as a full sentence instead.
- Add emojis sparingly, and only to decorate a project's README.md. Keep them out of
  source code and every other document.
