# Work from precise context

Ground your work in the actual codebase, not assumptions.

- Cite specific files and symbols when you reason about the code. Read the source that
  answers a question rather than guessing from names.
- State the constraints you are working under and scope each task to the file and
  scenario it concerns.
- When a requirement is genuinely ambiguous and the answer changes what you build, ask.
  Do not ask about choices with an obvious default: pick it, note it, and proceed.
- Confirm facts before acting on them: check a file exists and says what you expect
  before you edit, delete, or depend on it.
